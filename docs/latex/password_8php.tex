{\footnotesize\ttfamily const P\+A\+S\+S\+W\+O\+R\+D\+\_\+\+B\+C\+R\+Y\+PT \textquotesingle{}2y\textquotesingle{}}

The default algorithm to use for hashing if no algorithm is provided. This may change in newer P\+HP releases when newer, stronger hashing algorithms are supported. 

It is worth noting that over time this constant can (and likely will) change. Therefore you should be aware that the length of the resulting hash can change. Therefore, if you use {\bfseries{P\+A\+S\+S\+W\+O\+R\+D\+\_\+\+D\+E\+F\+A\+U\+LT}} you should store the resulting hash in a way that can store more than 60 characters (255 is the recomended width).

{\bfseries{P\+A\+S\+S\+W\+O\+R\+D\+\_\+\+B\+C\+R\+Y\+PT}} is used to create new password hashes using the {\bfseries{C\+R\+Y\+P\+T\+\_\+\+B\+L\+O\+W\+F\+I\+SH}} algorithm. 

This will always result in a hash using the \char`\"{}\$2y\$\char`\"{} crypt format, which is always 60 characters wide. 

Supported Options\+: 


\begin{DoxyItemize}
\item 

{\itshape salt} -\/ to manually provide a salt to use when hashing the password. Note that this will override and prevent a salt from being automatically generated. 



If omitted, a random salt will be generated by \mbox{\hyperlink{}{password\+\_\+hash()}} for each password hashed. This is the intended mode of operation. 


\item 

{\itshape cost} -\/ which denotes the algorithmic cost that should be used. Examples of these values can be found on the \mbox{\hyperlink{patch2_8txt_a3f9992ab8fa942ed02faa63682c3a84c}{crypt()"{}}} page. 



If ommitted, a default value of {\itshape 10} will be used. This is a good baseline cost, but you may want to consider increasing it depending on your hardware.
