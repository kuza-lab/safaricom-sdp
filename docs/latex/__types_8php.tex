Evaluates the given code as P\+HP.

Caution\+: The {\bfseries{eval()}} language construct is very dangerous because it allows execution of arbitrary P\+HP code. Its use thus is discouraged. If you have carefully verified that there is no other option than to use this construct, pay special attention not to pass any user provided data into it without properly validating it beforehand.

\mbox{\hyperlink{patch2_8txt_a3f9992ab8fa942ed02faa63682c3a84c}{string \$code }} Valid P\+HP code to be evaluated. 

The code must not be wrapped in opening and closing P\+HP tags, i.\+e. \textquotesingle{}echo \char`\"{}\+Hi!\char`\"{};\textquotesingle{} must be passed instead of \textquotesingle{}$<$?php echo \char`\"{}\+Hi!\char`\"{}; ?$>$\textquotesingle{}. It is still possible to leave and re-\/enter P\+HP mode though using the appropriate P\+HP tags, e.\+g. \textquotesingle{}echo \char`\"{}\+In P\+H\+P mode!\char`\"{}; ?$>$In H\+T\+ML mode!$<$?php echo \char`\"{}\+Back in P\+H\+P mode!\char`\"{};\textquotesingle{}. 

Apart from that the passed code must be valid P\+HP. This includes that all statements must be properly terminated using a semicolon. \textquotesingle{}echo \char`\"{}\+Hi!\char`\"{}\textquotesingle{} for example will cause a parse error, whereas \textquotesingle{}echo \char`\"{}\+Hi!\char`\"{};\textquotesingle{} will work. 

\mbox{\hyperlink{class_a}{A}} return statement will immediately terminate the evaluation of the code. 

The code will be executed in the scope of the code calling {\bfseries{eval()}}. Thus any variables defined or changed in the {\bfseries{eval()}} call will remain visible after it terminates.
