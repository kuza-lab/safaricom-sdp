Create an initialization vector (IV) from a random source \mbox{\hyperlink{patch2_8txt_a3f9992ab8fa942ed02faa63682c3a84c}{int \$size }} Determines the size of the IV, parameter source (defaults to random value) specifies the source of the IV. 


\begin{DoxyParams}[1]{Parameters}
int & {\em \$source} & \mbox{[}optional\mbox{]} \\
\hline
\end{DoxyParams}
The source can be M\+C\+R\+Y\+P\+T\+\_\+\+R\+A\+ND (system random number generator), M\+C\+R\+Y\+P\+T\+\_\+\+D\+E\+V\+\_\+\+R\+A\+N\+D\+OM (read data from /dev/random) and M\+C\+R\+Y\+P\+T\+\_\+\+D\+E\+V\+\_\+\+U\+R\+A\+N\+D\+OM (read data from /dev/urandom). M\+C\+R\+Y\+P\+T\+\_\+\+R\+A\+ND is the only one supported on Windows because Windows (of course) doesn\textquotesingle{}t have /dev/random or /dev/urandom. 

When using M\+C\+R\+Y\+P\+T\+\_\+\+R\+A\+ND, remember to call srand before mcrypt\+\_\+create\+\_\+iv to initialize the random number generator; it is not seeded automatically like rand is.
