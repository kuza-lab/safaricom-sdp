Generates a storable representation of a value. This is useful for storing or passing P\+HP values around without losing their type and structure. To make the serialized string into a P\+HP value again, use \mbox{\hyperlink{igbinary_8php_abb08bdf89d1fb8011add932e4b33562e}{igbinary\+\_\+unserialize}}.

\mbox{\hyperlink{igbinary_8php_adaa24700319ea052418e4844a4376d38}{igbinary\+\_\+serialize()}} handles all types, except the resource-\/type. You can even \mbox{\hyperlink{standard__4_8php_aa120148bb7608edd3ece62b379e5b8b4}{serialize()}} arrays that contain references to itself. Circular references inside the array/object you are \mbox{\hyperlink{standard__4_8php_aa120148bb7608edd3ece62b379e5b8b4}{serialize()}}ing will also be stored.

If object implements \mbox{\hyperlink{}{Serializable}} -\/interface, P\+HP will call the member function serialize to get serialized representation of object.

When serializing objects, P\+HP will attempt to call the member function \+\_\+\+\_\+sleep prior to serialization. This is to allow the object to do any last minute clean-\/up, etc. prior to being serialized. Likewise, when the object is restored using \mbox{\hyperlink{standard__4_8php_a9891e1553a8828910a805bb590e78a26}{unserialize()}} the \+\_\+\+\_\+wakeup member function is called.
