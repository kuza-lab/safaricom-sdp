Create an X\+ML parser \mbox{\hyperlink{patch2_8txt_a3f9992ab8fa942ed02faa63682c3a84c}{string \$encoding \mbox{[}optional\mbox{]} }} The optional {\itshape encoding} specifies the character encoding for the input/output in P\+HP 4. Starting from P\+HP 5, the input encoding is automatically detected, so that the {\itshape encoding} parameter specifies only the output encoding. In P\+HP 4, the default output encoding is the same as the input charset. If empty string is passed, the parser attempts to identify which encoding the document is encoded in by looking at the heading 3 or 4 bytes. In P\+HP 5.\+0.\+0 and 5.\+0.\+1, the default output charset is I\+S\+O-\/8859-\/1, while in P\+HP 5.\+0.\+2 and upper is U\+T\+F-\/8. The supported encodings are I\+S\+O-\/8859-\/1, U\+T\+F-\/8 and U\+S-\/\+A\+S\+C\+II.
