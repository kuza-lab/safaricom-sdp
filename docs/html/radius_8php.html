◆ radius_add_server()

radius_add_server (   $radius_handle,
  $hostname,
  $port,
  $secret,
  $timeout,
  $max_tries 
)

radius_add_server() may be called multiple times, and it may be used together with {

See also
radius_config()}. At most 10 servers may be specified. When multiple servers are given, they are tried in round-robin fashion until a valid response is received, or until each server's max_tries limit has been reached. resource $radius_handle string $hostname The hostname parameter specifies the server host, either as a fully qualified domain name or as a dotted-quad IP address in text form. int $port The port specifies the UDP port to contact on the server. If port is given as 0, the library looks up the radius/udp or radacct/udp service in the network services database, and uses the port found there.
If no entry is found, the library uses the standard Radius ports, 1812 for authentication and 1813 for accounting.
Parameters
string$secretThe shared secret for the server host is passed to the secret parameter. The Radius protocol ignores all but the leading 128 bytes of the shared secret.
int$timeoutThe timeout for receiving replies from the server is passed to the timeout parameter, in units of seconds.
int$max_triesThe maximum number of repeated requests to make before giving up is passed into the max_tries.
Returns
bool Returns TRUE on success or FALSE on failure.
See also
radius_config()
Since
1.1.0

◆ radius_config()

radius_config (   $radius_handle,
  $file 
)

Before issuing any Radius requests, the library must be made aware of the servers it can contact. The easiest way to configure the library is to call radius_config(). radius_config() causes the library to read a configuration file whose format is described in radius.conf. https://www.freebsd.org/cgi/man.cgi?query=radius.conf resource $radius_handle string $file The pathname of the configuration file is passed as the file argument to { radius_config()}. The library can also be configured programmatically by calls to { radius_add_server()}. bool Returns TRUE on success or FALSE on failure. radius_add_server() 1.1.0

A Radius request consists of a code specifying the kind of request, and zero or more attributes which provide additional information. To begin constructing a new request, call radius_create_request().
Note: Attention: You must call this function, before you can put any attribute! resource $radius_handle int $type Type is RADIUS_ACCESS_REQUEST or RADIUS_ACCOUNTING_REQUEST. bool Returns TRUE on success or FALSE on failure. radius_send_request() 1.1.0
